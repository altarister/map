# GDD: 레벨 1 - 챕터 1 (지명 외우기 / 지역 마스터)

## 0. 요청 
시니어기획자야 타야 그럼 지금 우리가 만드는것을 
1레벨: 지명외우기
1쳅터: 시군구 하나씩 마스터 
- 설정에서 1레벨 선택되어 있고 RegionSelectScreen창에서 쳅터를 선택할수 있는 ui 가 나오고 그중 1쳅터를 선택하면 하나만 선택할수 있는 시군구 리스트가 오고 그중 하를 선택하면 바로 게임에 진입한다. 
문제를 보고 지역을 클릭한다. 그렇게 모든 지역 소진되고 나면 결과 창이 나오는데. 그 결과 창에는 맞은겟수,  틀린겟수, 틀린지명들, 다시도전, 다른지역선택하기(만점이면 클릭가능) 요소들이 있다.. 어때 내가 뭐 빠진거 있나 
내가 만든 이름 바꾸지 말고 타임라인에 따른 요소들을 잘 정리해줘 

## 1. 개요 (Overview)
**목표**: 특정 시/군/구(예: 광주시) 예하의 모든 행정동을 완벽하게 암기할 때까지 반복 훈련.
**핵심 가치**: "한 놈만 팬다" (Focus & Mastery).
**타겟 유저**: 해당 지역의 지리를 처음 접하는 초보자.

## 2. 게임 흐름 (Game Flow)

### 2.1 진입 및 설정 (Entry & Configuration)
- **화면 구성**: `GameOptionSelectScreen` 내부에 게임 설정 기능을 통합합니다. (기존 `RegionSelectScreen` -> `GameOptionSelectScreen`으로 명칭 변경)
- **레벨 선택 (Level Selector)**:
  - **위치**: 화면 상단 중앙 타이틀 영역.
  - **방식**: `[ < Level 1: 지명 암기 > ]` 좌우 화살표나 탭 형태로 레벨을 직접 변경할 수 있습니다.
  - *변경점: 기존 `SettingsModal`에 있던 레벨 변경 기능을 이곳으로 가져옵니다.*
- **난이도 설정 (Training Options)**:
  - **위치**: 레벨 선택 하단 컨트롤 패널.
  - **스위치**: `Normal` (훈련) / `Hard` (실전) 토글.
  - **동작**:
    - **Normal**: 지도에 지명(RegionLabel) 표시.
    - **Hard**: 지명 숨김 (Blind Mode).

### 2.2 지역 선택 (Chapter Selection)
- **화면 메인 영역**: **"챕터 그리드(Chapter Grid)"**.
- **챕터 정의**: 경기도의 31개 시/군 = 31개의 챕터.
- **챕터 카드 (Chapter Card) UI**:
  - **구성**:
    - **타이틀**: 시/군 이름 (예: "광주시").
    - **볼륨**: 하위 행정동 개수 (예: "16개 동"). *(*구 단위가 아닌 동 단위)*
    - **숙련도**: `0%` ~ `100%` 숫자 표시.
    - **우선순위 표시**: 별도의 텍스트 태그(`🔥 필수`) 대신, **카드 테두리 색상**이나 **배경색(Highlight)**으로 은은하게 구분합니다. (개발자 재량)
- **선택 상호작용 (Single Select Interaction)**:
  - **초기 상태**: 모든 카드가 활성화되어 있습니다 (Opacity 1.0).
  - **하나 선택 시**:
    - 선택된 카드: `Active Style` (진한 테두리, 밝은 배경).
    - **나머지 카드**: `Dimmed` (Opacity 0.4) 처리되어 시각적으로 배제됩니다.
  - **교체 하기**:
    - 딤드된 다른 카드를 클릭하면 -> 기존 선택은 해제되고, 클릭한 카드가 `Active`가 되며 나머지는 다시 딤드됩니다.
    - *선택 해제(빈 곳 클릭)* 시 모든 카드가 다시 활성화됩니다.
  - **미션 시작**:
    - 카드를 선택하면 하단 `[MISSION START]` 버튼이 활성화됩니다.

### 2.3 게임 플레이 (Gameplay Loop)
- **진행 방식**:
  1.  **큐(Queue) 생성**: 선택한 시/군의 **모든 행정동**을 리스트에 담습니다.
  2.  **출제**: 랜덤으로 하나씩 문제를 냅니다.
  3.  **정답/오답**:
      - ✅ **정답**: 큐에서 제거.
      - ❌ **오답**: **큐에서 제거 안 됨.** 리스트의 뒤쪽으로 보내서 언젠가 다시 나오게 함.
  4.  **완료**: 큐가 완전히 빌 때까지(0) 계속됩니다.

### 2.4 결과 화면 (Result Screen)
- **화면 구성**: `ResultModal` 팝업.
- **표시 정보**:
  - **정답 개수 (Secured)**: (예: 16/16) - 전체 동 개수.
  - **오답 횟수 (Errors)**: (예: 5) - 총 몇 번 틀렸는지 누적.
  - **오답 노트 (Missed Targets)**: 틀린 지역들의 리스트를 보여줍니다.
- **액션 버튼 (Next Actions)**:
  1.  **[RETRY MISSION] (다시 도전)**:
      - 동일한 지역으로 즉시 재시작합니다.
      - *목표*: 오답 횟수(Errors)를 0으로 줄이는 것.
  2.  **[SELECT NEW SECTOR] (다른 지역 선택)**:
      - **활성화 조건**: 큐를 모두 비웠을 때 (Clear).
      - *기능*: 로비(`GameSelectScreen`)로 나가서 다른 챕터를 선택할 수 있습니다.
      - *마스터 표식*: 만약 오답이 0인 상태로 클리어했다면, 해당 챕터 카드에 "MASTERED" 마크가 붙습니다.

## 3. 기술적 요구사항 (Technical Requirements)

### 3.1 GameContext 수정
- **`queue` 시스템 도입**: 단순 인덱스(`currentIndex`) 증가 방식이 아닌, 남은 문제 리스트(`remainingQuestions`) 관리.
- **`retry` 로직**: 오답 시 해당 문제를 리스트의 뒤쪽(예: 3칸 뒤)에 재삽입 (Spaced Repetition).
- **`completion` 조건**: `remainingQuestions.length === 0`.

### 3.2 GameSelectScreen (구 RegionSelectScreen) 구현
- **컴포넌트 명**: `RegionSelectScreen` -> `GameSelectScreen`으로 변경 혹은 역할 확장.
- 레벨 선택 및 난이도 설정 UI 통합.
- "챕터 카드(Chapter Card)" 컴포넌트 분리 및 그리드 레이아웃.

### 3.3 ResultModal 수정
- 오답 노트 UI 추가.
- "다른 지역 선택" 버튼의 활성화/비활성화 로직 (Clear 여부 체크).
