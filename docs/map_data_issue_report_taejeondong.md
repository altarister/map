# Map Data Discrepancy Report: "태전동" (Taejeon-dong) Missing

## 1. Issue Description
지도를 렌더링할 때 경기도 광주시의 **"태전동"** 지역이 지도 상에 표시되지 않는 현상이 발견되었습니다.

## 2. Investigation Details
현재 프로젝트는 `public/data/skorea-submunicipalities-2018-geo.json` 파일 데이터를 기반으로 읍/면/동(Level 3) 단위의 지역 폴리곤을 그립니다. 
해당 JSON 데이터를 분석하여 경기도 광주시(행정구역 코드 31250)에 속한 하위 구역들을 조회해 본 결과, 다음과 같은 10개 구역만 존재함을 확인했습니다.

*   오포읍, 초월읍, 곤지암읍, 도척면, 퇴촌면, 남종면, 남한산성면, 경안동, 송정동, 광남동

**결론적으로 현재 지도의 원본 데이터 파일 안에 "태전동" 데이터 폴리곤이 아예 존재하지 않습니다.**

## 3. Root Cause Analysis
이 현상은 맵 데이터의 구조적 특징과 특수성 때문에 발생한 정상적인 동작의 결과입니다. 이유는 다음과 같습니다.

1.  **행정동 vs 법정동의 차이:**
    현재 프로젝트에 사용된 지도 데이터는 2018년 기준 **"행정동"** 단위로 파티션된 데이터입니다.
    "태전동"은 **"법정동"**이며, 해당 구역은 2018년 기준으로 행정동인 **"광남동"** 관할 구역에 포함되어 있었습니다. (현재 광남동은 광남1동, 2동으로 분동되었습니다.)
    따라서 지도 상에는 태전동 구역이 "광남동"이라는 하나의 큰 영역으로 합쳐져서 그려지고 있는 것입니다.
2.  **데이터 버전 (2018년):**
    사용 중인 데이터가 2018년 버전이므로, 최근에 변경된 행정 구역 분동 사항이 반영되어 있지 않습니다.

## 4. Conclusion & Action
요청하신 대로 소스코드 및 데이터는 어떠한 변경도 하지 않은 상태로 두었습니다. 
이 문제는 시스템의 버그가 아니라 **GeoJSON 데이터 파일의 기준이 '법정동'이 아닌 과거 '행정동' 기반이라 발생하는 문제**입니다. 추후 유저가 태전동과 같은 법정동을 명확하게 식별하고 학습하려면 두 가지 해결 방향을 고려할 수 있습니다.

*   **방안 A:** 최신(2023년 이후) '법정동' 기준의 GeoJSON 파일로 데이터를 교체 구축합니다.
*   **방안 B:** 게임의 룰을 '법정동' 맞추기가 아닌 '행정동' 맞추기임을 유저(플레이어)에게 명시합니다.
