### 0.2 핵심 데이터 구조 확인 (광역 자치단체/시·군·자치구/읍·면·법정동 계층 시스템)

우리의 지도 정책은 대한민국의 실제 통계청 행정구역 분류 체계(자릿수)를 그대로 따릅니다.

1. **광역 자치단체**: 앞 2자리 (`11` 서울, `23` 인천, `31` 경기 등)
2. **시/군/자치구**: 앞 5자리 (`11010` 종로구, `41110` 수원시 등)
3. **읍/면/법정동**: 7자리~8자리 (`1101053` 사직동 등)

`gyeonggi_bupjeongdong.geojson`의 properties 필드:

```json
{
  "code": "41610101",       ← 8자리: 읍/면/동 그룹 (퀴즈 출제 대상)
  "name": "경안동",
  "SIG_KOR_NM": "광주시",   ← 시군구 이름 (구 있는 시 판별 핵심 필드!)
  "EMD_KOR_NM": "경안동",
  "_isEmdGroup": true
}

{
  "code": "4161025025",    ← 10자리: 리(里) 단위 (출제 제외)
  "name": "산이리",
  "SIG_KOR_NM": "광주시"
}
```

### 0.3 행정구역 유형 자동 감지 — 구현 가능성: ✅ 완전 가능

**`SIG_KOR_NM` 필드의 공백(space) 패턴으로 100% 구별 가능:**

| 패턴 | 예시 | 해석 |
|------|------|------|
| `"SIG_KOR_NM": "광주시"` | 공백 없음 | 구 없는 시 |
| `"SIG_KOR_NM": "가평군"` | 공백 없음, `군`으로 끝남 | 군(郡) |
| `"SIG_KOR_NM": "용인시 처인구"` | 공백 있음, 뒤에 `구` | 구 있는 시 |

```typescript
// 완전히 구현 가능한 코드 (데이터에서 검증됨)
function detectAdminType(sigKorNm: string): AdminUnitType {
  if (sigKorNm.includes(' ')) return 'GU_CITY';  // "용인시 처인구"
  if (sigKorNm.endsWith('군')) return 'GUN';       // "가평군"
  return 'PLAIN_CITY';                              // "광주시"
}
```

## 1. 문제 정의 (Problem Statement)

### 1.1 핵심 충돌

기존 GDD의 코스 설계(기본 코스 / 상세 코스)는 **행정구역 진입 단계가 일정하다는 암묵적 가정** 위에 세워져 있습니다. 그러나 대한민국 행정구역은 도시마다 계층 단계가 다릅니다.

---

## 2. 대한민국 행정구역 통합 분류 체계 (Systematic Hierarchy)

대한민국의 행정구역은 다음과 같은 **3단계 구조**의 데이터 코드로 완벽하게 계층화되어 있으며, 게임도 이 위상을 따릅니다.

* **[광역 자치단체] (코드 2자리)**
  * 예: 서울(`11`), 인천(`23`), 경기(`31`)
  * **UI 취급**: 첫 진입 시 화면에 깔리는 가장 큰 땅입니다. (`seoul_incheon_level1.geojson`, `gyeonggi_level1_merged.geojson` 사용)

* **[시/군/자치구] (코드 5자리)**
  * 예: 강남구(`11230`), 부평구(`23060`), 용인시 기흥구(`31192`)
  * **핵심 예외 처리 (수원/용인/고양 등 대도시 "일반구" 문제)**
    * 데이터상 `skorea-municipalities-2018-geo.json` 파일에는 "용인시 전체"(`41460`) 폴리곤이 시/군/자치구 단위에 존재하지 **않습니다**.
    * 하지만 전국 스케일(경상북도, 전라남도 등) 확장을 고려했을 때, 포항시 남/북구나 전주시 완산/덕진구 등을 처음부터 평면(Flat)에 노출하면 **유저의 지리적 상식(도 -> 시 -> 구)과 크게 충돌**합니다.
    * 따라서 시스템 및 UI 관점에서는 쪼개진 원본 데이터(처인구, 기흥구)를 코드로 강제 병합하여 **"용인시" 전체를 대표하는 단일 [시/군/자치구] 단위로 묶어서 취급**합니다.
  * **UI 취급**: [광역 자치단체]의 땅을 누르면 나타나는 하위 구역. "기본 코스"의 무대.

* **[읍/면/법정동] (코드 7~8자리)**
  * 예: 역삼동(`1123064`), 산이리 등은 읍/면 그룹(`_isEmdGroup`)으로 병합.
  * **UI 취급**: 최종 퀴즈의 타겟들. (가장 말단이므로 리 단위는 영구 배제)

---

### 도시 유형별 화면 진입 해석 (전국구 유저 상식 우선 정책 - A안)

**전국구 확장을 위한 표준화 타겟**
대도시의 하위 일반구를 처음부터 화면에 쫙 뿌리면(Flat), 유저는 광역 자치단체 바로 밑에 생소한 구 이름들을 맞닥뜨려 혼란을 겪게 됩니다. 따라서 **대한민국의 정식 행정구역 체계(유저 상식)를 UI 화면 진입 순서로 그대로 적용**합니다.

**1) 서울 / 인천 / 부산 등 (특별·광역시 형)**
- [광역 자치단체] 클릭: "서울특별시" (11) 선택
- [시/군/자치구] 화면: 강남구, 서초구, 마포구 등 **분리된 자치구들이 한 화면에 렌더링**됨 (정상)
- [단일 구 클릭 시] 퀴즈: 강남구 진입 시 곧바로 "역삼동", "삼성동" 출제

**2) 용인 / 수원 / 포항 / 전주 등 (일반구가 있는 특례시·대도시 형 - 일반구 선택 화면 추가)**
- [광역 자치단체] 클릭: "경기도" (31) 
- [시/군/자치구] 화면: 기흥구, 처인구가 직접 노출되지 않고 묶여서 **"용인시" 단일 버튼/폴리곤으로 렌더링**됨. (포항시 역시 남/북구가 하나로 묶임)
- [단일 시 클릭 시] 상세 구역 선택창: "용인시 기흥구", "용인시 처인구" 중 하나를 고르는 **중간 구 선택 화면 삽입**
- [구 클릭 시] 퀴즈: 기흥구 진입 시 "보라동", "상갈동" 출제

**3) 광주 / 가평 / 구미 등 (구 없는 일반 시·군 형)**
- [광역 자치단체] 클릭: "경기도" (31) 
- [시/군/자치구] 화면: 광주시, 가평군 단일 노출
- [단일 시/군 클릭 시] 퀴즈: 하위 구가 없으므로 중간 자치구 선택 화면을 생략하고 곧바로 "경안동", "오포읍" 출제

## 3. 리(里) 효용성 최종 판단

### 3.1 도로명주소에서의 리(里) 위상

```
[도로명 주소 표준]
경기도 광주시 초월읍 경충대로 11번길 34   ← '산이리'는 주소에 없음
```

- 도로명주소(2014년 전면 시행): **리를 명시하지 않음**
- 화물 배차앱: 도로명주소 기반 → **리 단위 노출 안 됨**
- 화물 기사 판단에 읍/면 수준이면 충분

### 3.2 데이터 관점에서도 리는 불필요

- 경기도 데이터에서 10자리 코드(리 단위) 1,398개 존재
- 하지만 **8자리(읍/면/동 그룹)로 이미 병합 처리가 완료**되어  
  `_isEmdGroup: true` 플래그가 붙어 있음
- 현재 게임 로직도 이미 8자리 코드만 사용 중

### 3.3 결론: 리(里) 전면 제외 확정

> ✅ **데이터에도 이미 이 방향으로 구현되어 있음. 정책과 데이터가 일치.**

## 4. 코스 체계 (Course System)

### 4.1 퀴즈 출제 단위 확정

```
최종 퀴즈 출제 단위 = [읍/면/법정동]  (배차 플랫폼 노출 기준)
```

### 4.2 코스 정의 (전국 표준화 룰)

| 코스명 | 타겟 스케일 | 설명 |
|--------|-----------|-----------|
| **[A] 광역 코스** | 시/군/자치구 | 광역 자치단체 단계에서 소속 시/군/자치구 조각들을 맞추는 훈련 |
| **[B] 지역 코스** | 읍/면/법정동 | 시/군/자치구 하나를 찍고, 그 안의 모든 실전 배차지(읍/면/법정동)를 맞추는 심화 훈련. ★ 핵심 |

### 4.3 코스 선택 UI 흐름 및 성능 최적화 가이드

**개발 부하(DOM 폭발 및 Topo-gap)를 최소화하기 위한 렌더링 원칙**을 기획 단에서 강제합니다.

```
[1. 광역 자치단체 선택 화면] 광역 자치단체 (서울, 인천, 경기 등) 렌더링 (폴리곤 최소화, 부하 0)
         ↓
         원하는 광역 버튼(Ex. 경기도) 클릭
         ↓
[2. 시/군/자치구 선택 화면] 시/군/자치구 (렌더링 부하 억제)
         - 선택한 광역 단위 하위의 [시/군/자치구] 폴리곤만 화면에 렌더링함.
         - ⚠️ 개발 수칙 1: 일반구를 소유한 거대 도시(용인, 수원 등)는 구 단위 조각 폴리곤을 그대로 렌더링하지 말고, **무조건 "용인시" 1개 덩어리로 유니온(Union)하여 렌더링**함.
         ↓
         원하는 지역(Ex. 용인시) 클릭
         ↓
[3. 대도시 일반구 선택 화면] 일반구 선택 (⚠️ 대도시 전용)
         - 용인시의 하위 구역인 기흥구, 처인구, 수지구를 선택하는 중간 모달/화면 노출.
         - (가평군처럼 하위 일반구가 없는 곳은 본 선택 화면을 생략하고 곧바로 퀴즈로 직행함)
         ↓
         원하는 기흥구 클릭
         ↓
[4. 읍/면/법정동 퀴즈 화면] (이빨 빠짐 방지 및 렌더링 최적화)
         - 기흥구에 소속 20여 개 동(읍/면/법정동) 폴리곤만 렌더링함.
         - ⚠️ 개발 수칙 2: 줌아웃해서 전국 3,500개 읍면동을 한 번에 그리는 기능은 기획적으로 금지됨. 항상 타겟 지역 내부만 렌더링함.
```
