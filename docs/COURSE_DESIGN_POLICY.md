### 0.2 핵심 데이터 구조 확인 (Level 계층 시스템)

우리의 지도 정책은 대한민국의 실제 통계청 행정구역 분류 체계(자릿수)를 그대로 따릅니다.

1. **Level 1 (광역 자치단체)**: 앞 2자리 (`11` 서울, `23` 인천, `31` 경기 등)
2. **Level 2 (기초 자치단체)**: 앞 5자리 (`11010` 종로구, `41110` 수원시 등)
3. **Level 3 (읍/면/동)**: 7자리~8자리 (`1101053` 사직동 등)

`gyeonggi_bupjeongdong.geojson`의 properties 필드:

```json
{
  "code": "41610101",       ← 8자리: 읍/면/동 그룹 (퀴즈 출제 대상)
  "name": "경안동",
  "SIG_KOR_NM": "광주시",   ← 시군구 이름 (구 있는 시 판별 핵심 필드!)
  "EMD_KOR_NM": "경안동",
  "_isEmdGroup": true
}

{
  "code": "4161025025",    ← 10자리: 리(里) 단위 (출제 제외)
  "name": "산이리",
  "SIG_KOR_NM": "광주시"
}
```

### 0.3 행정구역 유형 자동 감지 — 구현 가능성: ✅ 완전 가능

**`SIG_KOR_NM` 필드의 공백(space) 패턴으로 100% 구별 가능:**

| 패턴 | 예시 | 해석 |
|------|------|------|
| `"SIG_KOR_NM": "광주시"` | 공백 없음 | 구 없는 시 |
| `"SIG_KOR_NM": "가평군"` | 공백 없음, `군`으로 끝남 | 군(郡) |
| `"SIG_KOR_NM": "용인시 처인구"` | 공백 있음, 뒤에 `구` | 구 있는 시 |

```typescript
// 완전히 구현 가능한 코드 (데이터에서 검증됨)
function detectAdminType(sigKorNm: string): AdminUnitType {
  if (sigKorNm.includes(' ')) return 'GU_CITY';  // "용인시 처인구"
  if (sigKorNm.endsWith('군')) return 'GUN';       // "가평군"
  return 'PLAIN_CITY';                              // "광주시"
}
```

## 1. 문제 정의 (Problem Statement)

### 1.1 핵심 충돌

기존 GDD의 코스 설계(기본 코스 / 상세 코스)는 **행정구역 깊이(Depth)가 일정하다는 암묵적 가정** 위에 세워져 있습니다. 그러나 대한민국 행정구역은 도시마다 계층 깊이가 다릅니다.

---

## 2. 대한민국 행정구역 통합 분류 체계 (Systematic Hierarchy)

대한민국의 행정구역은 다음과 같은 **3-Level 구조**의 데이터 코드로 완벽하게 계층화되어 있으며, 게임도 이 위상을 따릅니다.

* **[Level 1] 특별시 / 광역시 / 도 (코드 2자리)**
  * 예: 서울(`11`), 인천(`23`), 경기(`31`)
  * **UI 취급**: 첫 진입 시 화면에 깔리는 가장 큰 땅입니다. (`seoul_incheon_level1.geojson`, `gyeonggi_level1_merged.geojson` 사용)

* **[Level 2] 시 / 군 / 자치구 (코드 5자리)**
  * 예: 강남구(`11230`), 부평구(`23060`), 용인시 기흥구(`31192`)
  * **핵심 예외 처리 (수원/용인/고양 등 대도시 "비자치구" 문제)**
    * 데이터상 `skorea-municipalities-2018-geo.json` 파일에는 "용인시 전체"(`41460`) 폴리곤이 Level 2에 존재하지 **않습니다**.
    * 대신 그 자리를 "용인시 처인구"(`31191`), "용인시 기흥구"(`31192`), "용인시 수지구"(`31193`)가 **서울의 '종로구'와 동일한 Level 2 위상**으로 차지하고 조각나 있습니다.
    * 따라서 시스템 관점에서는 "용인시"라는 L2 단위는 없으며, **"기흥구" 자체가 서울의 "강남구"와 완벽히 동급인 Level 2로 취급**됩니다.
  * **UI 취급**: [Level 1]의 땅을 누르면 나타나는 하위 구역. "기본 코스"의 무대.

* **[Level 3] 읍 / 면 / 동 (코드 7~8자리)**
  * 예: 역삼동(`1123064`), 산이리 등은 읍/면 그룹(`_isEmdGroup`)으로 병합.
  * **UI 취급**: [Level 2]를 선택했을 때 출제되는 **최종 퀴즈의 타겟들**. (가장 말단이므로 리 단위는 영구 배제)

---

### 도시 유형별 Depth 해석 (L1.5 가상 묶음 폐지 및 플랫화)

**기존 기획의 한계 (알의 지적 사항 반영)**
대도시는 "용인시"로 한 번 묶어서(L1.5) 보여주려 했으나, 이는 UI 뎁스만 깊어지게 만들고 시스템 복잡도를 가중시킵니다. 실제로 콜을 잡는 기사님들은 "기흥구", "처인구" 자체를 독립된 인지 구역으로 다룹니다. 따라서 **어떤 도시든 예외 없이 화면에 쫙 뿌려주는 Flat한 선택창**을 채택합니다.

**1) 서울 / 인천 (특별·광역시 형)**
- [L1] 클릭: "서울특별시" (11) 선택
- [L2] 화면: 강남구, 서초구, 마포구 등 **25개 구가 한 화면에 렌더링**됨
- [L3] 퀴즈: 강남구 진입 시 "역삼동", "삼성동" 출제

**2) 용인 / 수원 (특례시·대도시 형 - '용인시' 가상 폴더링 폐지)**
- [L1] 클릭: "경기도" (31) 지도에서 해당 위치 클릭
- [L2] 화면: 용인시 처인구, 기흥구, 수지구를 포함한 **경기도 내 42개 시/군/구가 모두 한 화면에 평등하게 렌더링**됨 (폴더 뎁스 없음)
- [L3] 퀴즈: 기흥구 진입 시 "보라동", "상갈동" 출제

**3) 광주 / 가평 (중소도시·군 형)**
- [L1] 클릭: "경기도" (31) 
- [L2] 화면: 광주시, 가평군 역시 42개 리스트 중 하나로 평등하게 존재
- [L3] 퀴즈: 광주시 진입 시 "경안동", "오포읍" 출제

## 3. 리(里) 효용성 최종 판단

### 3.1 도로명주소에서의 리(里) 위상

```
[도로명 주소 표준]
경기도 광주시 초월읍 경충대로 11번길 34   ← '산이리'는 주소에 없음
```

- 도로명주소(2014년 전면 시행): **리를 명시하지 않음**
- 화물 배차앱: 도로명주소 기반 → **리 단위 노출 안 됨**
- 화물 기사 판단에 읍/면 수준이면 충분

### 3.2 데이터 관점에서도 리는 불필요

- 경기도 데이터에서 10자리 코드(리 단위) 1,398개 존재
- 하지만 **8자리(읍/면/동 그룹)로 이미 병합 처리가 완료**되어  
  `_isEmdGroup: true` 플래그가 붙어 있음
- 현재 게임 로직도 이미 8자리 코드만 사용 중

### 3.3 결론: 리(里) 전면 제외 확정

> ✅ **데이터에도 이미 이 방향으로 구현되어 있음. 정책과 데이터가 일치.**

## 4. 코스 체계 (Course System)

### 4.1 퀴즈 출제 단위 확정

```
최종 퀴즈 출제 단위 = [Level 3] 읍 | 면 | 동  (배차 플랫폼 노출 기준)
```

### 4.2 코스 정의 (전국 표준화 룰)

| 코스명 | 타겟 스케일 | 설명 |
|--------|-----------|-----------|
| **[A] 광역 코스** | Level 2 (시/군/구) | 큰 도 단위(L1)에서 시/군/구 조각들을 맞추는 훈련 |
| **[B] 지역 코스** | Level 3 (읍/면/동) | 시/군/구(L2) 하나를 찍고, 그 안의 모든 실전 배차지(읍면동)를 맞추는 심화 훈련. ★ 핵심 |

### 4.3 코스 선택 UI 흐름 및 성능 최적화 가이드

**개발 부하(DOM 폭발 및 Topo-gap)를 최소화하기 위한 렌더링 원칙**을 기획 단에서 강제합니다.

```
[초기 화면] Level 1 (서울, 인천, 경기 등) 렌더링 (폴리곤 3개, 부하 0)
         ↓
         원하는 지역 버튼(Ex. 서울특별시) 클릭
         ↓
[지역 화면] Level 2 (렌더링 부하 억제)
         - 전국을 그리지 않음. 방금 선택한 서울시 소속 25개 구(Level 2)의 폴리곤만 화면에 렌더링함. (폴리곤 25개, 부하 0)
         ↓
[퀴즈 돌입] Level 3 (이빨 빠짐 방지 및 렌더링 최적화)
         - 강남구를 선택했다면, 강남구 소속 20여 개 동(Level 3) 폴리곤만 렌더링함.
         - ⚠️ 개발 수칙 1: 이때 바닥에 깔던 **'강남구 전체 테두리(Level 2)' 레이어는 아예 그리지 않고 버림!** 오직 Level 3 레이어들만 조립해서 그림. (Topo-gap 근본적 차단)
         - ⚠️ 개발 수칙 2: 줌아웃해서 전국 3,500개 읍면동을 한 번에 그리는 기능은 기획적으로 금지됨. 항상 타겟 Level 2 영역 내부만 렌더링함.
```

## 5. 클릭 불가 지역 처리

```typescript
const MIN_CLICKABLE_AREA_PX2 = 2500; // 약 50×50px

// 퀴즈 풀에서 너무 작은 지역 제외
const quizPool = features.filter(f =>
  f.properties.code.length === 8 &&
  f.screenArea >= MIN_CLICKABLE_AREA_PX2
);
```

- 제외된 지역도 지도에는 표시 (반투명)
- 클릭 시 "이 지역은 퀴즈 출제 제외 구역" 토스트 표시
