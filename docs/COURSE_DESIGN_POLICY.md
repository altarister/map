### 0.2 핵심 데이터 구조 확인

`gyeonggi_bupjeongdong.geojson`의 properties 필드:

```json
{
  "code": "41610101",       ← 8자리: 읍/면/동 그룹 (퀴즈 출제 대상)
  "name": "경안동",
  "SIG_KOR_NM": "광주시",   ← 시군구 이름 (구 있는 시 판별 핵심 필드!)
  "EMD_KOR_NM": "경안동",
  "_isEmdGroup": true
}

{
  "code": "4161025025",    ← 10자리: 리(里) 단위 (출제 제외)
  "name": "산이리",
  "SIG_KOR_NM": "광주시"
}
```

### 0.3 행정구역 유형 자동 감지 — 구현 가능성: ✅ 완전 가능

**`SIG_KOR_NM` 필드의 공백(space) 패턴으로 100% 구별 가능:**

| 패턴 | 예시 | 해석 |
|------|------|------|
| `"SIG_KOR_NM": "광주시"` | 공백 없음 | 구 없는 시 |
| `"SIG_KOR_NM": "가평군"` | 공백 없음, `군`으로 끝남 | 군(郡) |
| `"SIG_KOR_NM": "용인시 처인구"` | 공백 있음, 뒤에 `구` | 구 있는 시 |

```typescript
// 완전히 구현 가능한 코드 (데이터에서 검증됨)
function detectAdminType(sigKorNm: string): AdminUnitType {
  if (sigKorNm.includes(' ')) return 'GU_CITY';  // "용인시 처인구"
  if (sigKorNm.endsWith('군')) return 'GUN';       // "가평군"
  return 'PLAIN_CITY';                              // "광주시"
}
```

## 1. 문제 정의 (Problem Statement)

### 1.1 핵심 충돌

기존 GDD의 코스 설계(기본 코스 / 상세 코스)는 **행정구역 깊이(Depth)가 일정하다는 암묵적 가정** 위에 세워져 있습니다. 그러나 대한민국 행정구역은 도시마다 계층 깊이가 다릅니다.

---

## 2. 대한민국 행정구역 전체 유형 (+ 데이터 대응)

### 2.1 유형 A: 특별시 — 서울

```
서울특별시 (코드 11)
├── 강남구 (11230) → 역삼동, 삼성동 ...
└── (총 25 자치구)
```

- `skorea-municipalities-2018-geo.json`에 25개 자치구 존재 ✅
- `skorea-submunicipalities-2018-geo.json`에 동 단위 존재 ✅
- **읍·면·리 없음**: 동이 최하위 단위
- **지역 코스만** 제공 (구 선택 → 동 출제)

### 2.2 유형 B: 광역시 — 부산, 대구, 인천, 광주, 대전, 울산

- 자치구(구): 구 → 동 (리 없음) → 지역 코스만
- 군(郡): 군 → 읍/면 → (리) → 지역 코스 (리 제외)
- 예: 부산 기장군(읍/면 있음), 울산 울주군

### 2.3 유형 C: 도 소속 대도시 — 수원, 용인, 고양 (경기도 내 확인됨 ✅)

- 데이터에서 `SIG_KOR_NM`이 `"수원시 장안구"` 형태 → 공백으로 자동 감지
- 광역 코스 + 지역 코스 2단계 제공

### 2.4 유형 D: 도 소속 중소시 — 광주시, 여주시 (경기도 내 확인됨 ✅)

- 데이터에서 `SIG_KOR_NM`이 `"광주시"` 단일 형태
- 지역 코스만 제공

### 2.5 유형 E: 군(郡) — 가평군, 양평군, 연천군 (경기도 내 확인됨 ✅)

- 데이터에서 `SIG_KOR_NM`이 `"가평군"` 형태, `군`으로 끝남
- 동(洞) 없이 읍/면만 존재
- 지역 코스만 제공 (단, 읍/면 수가 적음: 가평군 6개, 연천군 10개)

## 3. 리(里) 효용성 최종 판단

### 3.1 도로명주소에서의 리(里) 위상

```
[도로명 주소 표준]
경기도 광주시 초월읍 경충대로 11번길 34   ← '산이리'는 주소에 없음
```

- 도로명주소(2014년 전면 시행): **리를 명시하지 않음**
- 화물 배차앱: 도로명주소 기반 → **리 단위 노출 안 됨**
- 화물 기사 판단에 읍/면 수준이면 충분

### 3.2 데이터 관점에서도 리는 불필요

- 경기도 데이터에서 10자리 코드(리 단위) 1,398개 존재
- 하지만 **8자리(읍/면/동 그룹)로 이미 병합 처리가 완료**되어  
  `_isEmdGroup: true` 플래그가 붙어 있음
- 현재 게임 로직도 이미 8자리 코드만 사용 중

### 3.3 결론: 리(里) 전면 제외 확정

> ✅ **데이터에도 이미 이 방향으로 구현되어 있음. 정책과 데이터가 일치.**

## 4. 코스 체계 (Course System)

### 4.1 퀴즈 출제 단위 확정

```
최종 퀴즈 출제 단위 = 읍 | 면 | 동  (8자리 코드 기준)
```

### 4.2 코스 정의

| 코스명 | 출제 단위 | 제공 조건 |
|--------|-----------|-----------|
| **광역 코스** | 구(區) | 구 있는 시만 (SIG_KOR_NM에 공백) |
| **지역 코스** | 읍/면/동 | 모든 지역 ★핵심 |

### 4.3 도시 유형별 코스 매트릭스 (데이터 기반)

| 행정 유형 | 경기도 예시 | 데이터 감지 방법 | 광역 코스 | 지역 코스 |
|-----------|-------------|-----------------|-----------|-----------|
| 구 있는 시 | 용인시, 수원시 | SIG_KOR_NM 공백 포함 | ✅ 구 출제 | ✅ 구 선택 후 읍/면/동 |
| 구 없는 시 | 광주시, 이천시 | SIG_KOR_NM 공백 없음, 시 | ❌ | ✅ 읍/면/동 |
| 군(郡) | 가평군, 양평군 | SIG_KOR_NM '군'으로 끝 | ❌ | ✅ 읍/면 |
| 서울 자치구 | 강남구 | code 시작 '11' | ❌ | ✅ 동 |
| 광역시 자치구 | 해운대구 | code 시작 21~29 | ❌ | ✅ 동 |

### 4.4 코스 선택 UI 흐름 (구현 가이드)

```
[1단계] 시/군 선택
        ↓
  SIG_KOR_NM 공백 여부로 자동 감지
        ↓
  ┌─ 구 있는 시 → 구 목록 표시 → [광역 코스] → 구 선택
  └─ 구 없는 시/군 → 바로 [지역 코스] 시작

[2단계] [지역 코스] 시작
  features.filter(f => f.properties.code.length === 8)
         를 퀴즈 풀로 사용
```

## 5. 클릭 불가 지역 처리

```typescript
const MIN_CLICKABLE_AREA_PX2 = 2500; // 약 50×50px

// 퀴즈 풀에서 너무 작은 지역 제외
const quizPool = features.filter(f =>
  f.properties.code.length === 8 &&
  f.screenArea >= MIN_CLICKABLE_AREA_PX2
);
```

- 제외된 지역도 지도에는 표시 (반투명)
- 클릭 시 "이 지역은 퀴즈 출제 제외 구역" 토스트 표시
